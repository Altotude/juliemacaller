A) HTML Structure - top

This theme is a fixed layout with one column. All of the information within the main content area is nested within a div with an id of "container main". The general template structure is the same throughout the template. Here is the general structure.

 

If you would like to edit the color, font, or style of any elements in one of these columns or if you want to change the background picture, you would do the following:

body{
	background:url('pathtobackgroundpictureHERE');
	font-family: FontHERE;
	color:#hexcolorhere;
}
a{
    color:#hexcolorhere;

}

If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.

.menu ul li a:hover,.menu ul li a.selected{
    color:#569099;
    border-bottom: 2px solid #569099;
    text-decoration: none;
}

So, to ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours. Classes are most used in the CSS. All the HTML code is commented in the most important points of it.

B) CSS Files and Structure - top

I'm using two CSS files in this theme. The first one is Twitter Bootstrap with generic reset content included. Many browser interpret the default behavior of html elements differently. By using a general reset CSS file and a base skeleton to build upon, we can work round this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

The second file contains all of the specific stylings for the page. The file is separated into sections using:

	/* === Header Section === */

	/* === Content Section === */
	
	/* === Sidebar Section === */
	
	/* === Footer === */

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

Any images that are placed within the blog section have 5px worth of padding and a light bluish background. If you would like to edit the display of these images, find the following section in the style sheet:

	.content .about-content img {
		change styles here:
	}

C) JavaScript - top

This theme imports three Javascript files.

  1. jQuery
  2. Cufon
  3. My custom scripts
  4. FancyBox
  5. Quicksand
  1. jQuery is a Javascript library that greatly reduces the amount of code that you must write.
  2. Cufon is a javascript file that has the sole purpose of integrating unsuported special or custom fonts into any web design using only javascript and html.
  3. Most of the animation in this site is carried out from the customs scripts. There are a few functions worth looking over. There are some sctions that handle the portfolio menu hovering and setting it's class on click. That is something that can also be found in script.js custom script file.
  4. Fancybox is only used in the portfolio section of the template and has the purpose of showing a light-box pop-ul for each photo in the portfolio section.
  5. Quicksand is a jQuery plugin that creates a slide, change and fade effect for portfolio items.

In addition to the custom scripts, I implement a few "tried and true" plugins to create the effects. This plugin is packed, so you won't need to manually edit anything in the file. The only necessary thing to know is how to call the method. For example: 

	
            $('.customers').tinyscrollbar({
        size:88
    });

This section will create the functionality of the little scrollbar content in the bottom right.
